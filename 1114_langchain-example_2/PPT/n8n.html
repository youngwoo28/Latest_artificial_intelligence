n8n 이란? (What is n8n?)

노드(Node) 기반 워크플로우 자동화 도구입니다.

다양한 웹 서비스와 애플리케이션을 코딩 없이 시각적으로 연결하여, 반복적인 작업을 자동화하고 복잡한 프로세스를 구축할 수 있게 도와줍니다.

마치 디지털 레고(LEGO)처럼, 각각의 기능(노드)을 연결하여 원하는 자동화 시스템을 만드는 것과 같습니다.

n8n의 라이선스: Fair-Code

n8n은 엄밀한 의미의 오픈소스(OSI Preamble)가 아닌 "Source-Available" 소프트웨어입니다.

소스 코드는 공개되어 있지만, 상업적 재판매나 n8n의 클라우드 버전과 직접 경쟁하는 서비스를 만드는 것에는 제한이 있는 Fair-Code 라이선스를 따릅니다.

핵심: 개인 및 기업 내부용으로 자체 호스팅(Self-hosting)하여 사용하는 것은 완전히 무료입니다.

경쟁 서비스 비교: n8n vs Zapier

n8n

  • 비용: 자체 호스팅 시 거의 무료
  • 데이터 소유권: 내 서버에 저장 (높은 보안)
  • 유연성: 코드 작성, 복잡한 분기 처리 등 개발자에게 유리
  • 학습 곡선: 다소 높음 (특히 설치)

Zapier

  • 비용: 작업량에 따라 증가 (상대적 고비용)
  • 데이터 소유권: Zapier 서버에 저장
  • 유연성: GUI 기반으로 단순하지만, 복잡한 로직 구현에 한계
  • 학습 곡선: 매우 낮음 (초보자에게 친화적)

n8n의 장점과 단점

장점 (Pros)

  • 비용 효율성: 직접 서버에 설치 시 매우 저렴하게 운영 가능
  • 데이터 주권 확보: 민감한 데이터를 외부에 노출하지 않고 관리
  • 높은 자유도: IF, Switch, 코드 노드로 복잡한 워크플로우 설계 가능
  • 활발한 커뮤니티: 문제 해결 및 정보 공유가 용이

단점 (Cons)

  • 설치 및 유지보수: 자체 호스팅 시 서버 관리에 대한 지식 필요
  • 학습 난이도: Zapier 등 경쟁 서비스에 비해 초기 진입 장벽 존재
  • 상대적으로 적은 통합 앱: 주요 서비스는 대부분 지원하지만, Zapier만큼 다양하진 않음

Docker Deep Dive

A Closer Look for CS Majors

Virtualization: VM vs Container

Docker는 OS-level 가상화 기술입니다. Hypervisor 기반의 전통적인 VM과 비교해 봅시다.

Docker Architecture & Core Concepts

1. Docker Engine: Client-Server 구조로 동작합니다.

  • Docker Client: 사용자가 docker run 등의 명령을 입력하는 CLI.
  • Docker Daemon (dockerd): Client의 요청을 받아 이미지, 컨테이너, 네트워크 등을 관리하는 백그라운드 프로세스.
  • REST API: Client가 Daemon과 통신하기 위해 사용하는 인터페이스.

2. Image: Union File System (e.g., OverlayFS)을 사용하는 Read-Only 레이어들의 집합입니다. Dockerfile에 정의된 순서대로 각 레이어가 쌓여 하나의 이미지를 구성합니다.

3. Container: 이미지의 Read-Only 레이어 위에 Read-Write 가능한 컨테이너 레이어를 추가하여 생성됩니다. 격리된 프로세스로서 Host의 커널 위에서 직접 실행됩니다.

4. Registry: Docker 이미지를 저장하고 공유하는 원격 저장소입니다. (e.g., Docker Hub, GCP AR, AWS ECR)

Windows & Docker

n8n 로컬 환경 실행 가이드

사전 준비 사항

  • 운영체제: Windows 10 또는 11 (Pro, Enterprise, Education 권장)
  • WSL 2: Windows Subsystem for Linux 2 기능이 활성화되어 있어야 합니다.
  • Docker Desktop: PC에 최신 버전의 Docker Desktop이 설치 및 실행 중이어야 합니다.
  • 터미널: PowerShell 또는 Windows Terminal과 같은 명령줄 도구가 필요합니다.

Docker Desktop은 내부적으로 WSL 2를 사용하여 Linux 컨테이너를 실행하므로, WSL 2 설치는 필수입니다.

1단계: 데이터 폴더 생성

n8n의 워크플로우, 자격 증명 등 모든 데이터를 PC에 영구적으로 저장하기 위한 폴더를 생성합니다. 이 폴더는 Docker 컨테이너에 연결(마운트)되어 데이터 유실을 방지합니다.

# PowerShell 또는 Windows Terminal을 실행합니다.
# 사용자 홈 디렉토리로 이동합니다.
cd ~

# n8n 데이터를 저장할 .n8n 폴더를 생성합니다.
mkdir .n8n

2단계: n8n 컨테이너 실행

아래 명령어를 터미널에 복사하여 붙여넣고 실행합니다. n8n Docker 이미지를 다운로드하고 컨테이너를 백그라운드에서 실행시킵니다.

docker run -d --name n8n -p 5678:5678 -v %USERPROFILE%\.n8n:/home/node/.n8n n8nio/n8n

-d: Detached mode (백그라운드)

--name n8n: 컨테이너 이름 지정

-p 5678:5678: 포트 포워딩 (Host:Container)

-v ...: 볼륨 마운트 (Host:Container)

3단계: n8n 접속 및 설정

컨테이너가 정상적으로 실행되면 웹 브라우저를 열고 아래 주소로 접속합니다.

http://localhost:5678

처음 접속 시 계정 설정을 위한 화면이 나타납니다. 안내에 따라 관리자 계정을 생성하면 n8n을 사용할 준비가 완료됩니다.

Docker 컨테이너 관리

Docker Desktop 대시보드 또는 터미널 명령어를 통해 n8n 컨테이너를 관리할 수 있습니다.

# 컨테이너 실행 로그 확인
docker logs n8n

# 컨테이너 중지
docker stop n8n

# 컨테이너 다시 시작
docker start n8n

# 컨테이너 삭제 (주의: 볼륨은 삭제되지 않음)
docker rm n8n
